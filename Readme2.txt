[РУССКАЯ ВЕРСИЯ]

1) Цель проекта
---------------
Цель дипломного проекта - цифровизация учебного процесса по робототехнике и ускорение обратной связи между преподавателем и студентом.
Платформа позволяет:
- преподавателю создавать и управлять заданиями;
- студенту сдавать проект;
- системе делать первичную AI-оценку;
- преподавателю выставлять итоговую оценку;
- вести обсуждение через комментарии.

2) Роли и предметная область
-----------------------------
В системе 2 ключевые роли:
- Teacher: создание заданий, проверка работ, аналитика;
- Student: просмотр заданий, отправка проекта, получение фидбека.

Ключевые сущности:
- User
- Assignment
- Project
- Rating
- Comment

3) Технологический стек
-----------------------
Frontend:
- React 18
- Vite
- TypeScript
- Tailwind CSS
- React Router
- Axios
- i18next (казахский, русский, английский)

Backend:
- Node.js
- Express.js
- MongoDB
- Mongoose
- JWT
- Multer
- Zod

AI:
- Gemini API + fallback логика

4) Архитектура
--------------
Использована layered-архитектура:
- routes (маршруты)
- controllers (бизнес-логика)
- models (схемы)
- middlewares (auth/role/error)
- utils/services (вспомогательная логика и интеграции)

5) Безопасность
---------------
- JWT-аутентификация
- RBAC (ограничение по ролям)
- Teacher-only endpoints
- Единый формат ошибок
- Безопасные имена файлов
- UUID в имени файла для предотвращения перезаписи

6) Основной функционал
----------------------
A) Модуль преподавателя:
- создание/редактирование/архивация/удаление заданий
- просмотр студентов
- аналитика
- удаление студента с каскадной очисткой данных

B) Модуль студента:
- просмотр заданий
- сдача проекта
- просмотр результатов и комментариев
- скачивание приложенных файлов

C) Модуль оценивания:
- AI-оценка
- ручная оценка преподавателя
- итоговый score

7) Работа с файлами
-------------------
- загрузка материалов и проектных файлов
- валидация форматов и размеров
- стабильная обработка ошибок

8) Темизация (Dark/Light)
-------------------------
- глобальный ThemeContext
- хранение выбора в localStorage
- единообразный UI во всех страницах

9) Локализация
--------------
- интерфейс на 3 языках
- переключение языка без перезагрузки

10) Производительность и устойчивость
-------------------------------------
- async/await + loading states
- защита от падений UI
- production build проверки
- предсказуемый API-контракт

11) Целостность данных
----------------------
При удалении студента удаляются связанные сущности (проекты, рейтинги, комментарии, файлы), чтобы не оставались orphan-данные.

12) Развертывание
-----------------
- Frontend: `vite build`
- Backend: Node.js runtime
- MongoDB Atlas
- постоянное хранилище для uploads

13) Итог
--------
RoboLearn - это полноценный full-stack веб-портал для образовательного процесса в робототехнике: с ролями, заданиями, проверкой, AI-поддержкой, локализацией и стабильной архитектурой.

14) Короткий текст для защиты
-----------------------------

Проект реализован на React + Node.js + MongoDB, с JWT-авторизацией и ролевой моделью.
